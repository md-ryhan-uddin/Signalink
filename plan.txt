ðŸŸ¦ ARCHITECTURE DESCRIPTION

Signalink follows a modern, distributed, event-driven architecture:

Frontend
Can be a lightweight React client or Postman tests
Connects via WebSockets (real-time) and REST API (management)

API Gateway
AWS API Gateway routes requests to backend services
Handles authentication, TLS, and optional throttling

FastAPI Services
REST API: authentication, channel management, message history
WebSocket server: real-time message delivery and presence management

Redis (Pub/Sub)
Used for ephemeral message broadcasting to connected clients
Stores presence info (online/offline users)

Kafka (MSK Serverless)
Core distributed event bus
Handles messages, notifications, analytics events

Durable, ordered event processing
PostgreSQL
Canonical message persistence
Stores users, channels, messages, read receipts

Analytics Microservice
Consumes Kafka streams to compute metrics (messages/sec, active users, per-channel stats)
Notification Service
Background worker to trigger offline notifications based on Kafka/Redis events

Observability
CloudWatch (metrics/logs)
Prometheus/Grafana 
Tracks latency, throughput, consumer lag, errors
Deployment
Dockerized services for reproducibility
Free-tier AWS deployment (EC2 t2.micro, API Gateway, MSK Serverless, S3)
GitHub Actions for CI/CD automation
